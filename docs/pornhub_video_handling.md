# How PornHub Video Handling Works in This Codebase

This document describes **precisely** how PornHub videos are handled in the repository, as implemented in the code.

---

## 1. Where PornHub Is Used

- **`main.py`** – Web app (FastAPI) uses PornHub for the “PornHub” workflow: Chrome, login, then **yt-dlp** to extract playlist URLs and download.
- **`archive/ph.py`** – Standalone script: **Firefox**, login, Netscape cookies, **Selenium + HTML parsing** for video URL extraction from the favorites page, then **yt-dlp** for per-video download (because yt-dlp returns 404 on the favorites playlist URL).

Both rely on **yt-dlp** with a **Netscape cookie file** for the actual video downloads.

---

## 2. URL Shapes (PornHub)

- **Video page (canonical):**  
  `https://www.pornhub.com/view_video.php?viewkey=<viewkey>`  
  Example: `https://www.pornhub.com/view_video.php?viewkey=ph5fe50e391efeb`

- **Favorites playlist (user-specific):**  
  `https://www.pornhub.com/users/<username>/videos/favorites`  
  Example: `https://www.pornhub.com/users/burninglove999/videos/favorites`  
  Pagination: `.../favorites?page=2`, etc. See [pornhub_favourites_page_and_pagination.md](pornhub_favourites_page_and_pagination.md).

- In HTML, video links appear as `<a href="...">` with:
  - `href` containing `view_video` or `viewkey=` (relative or absolute).

## 3. How `archive/ph.py` Handles PornHub (Precise)

1. **Browser**
   - **Firefox** via Selenium, `headless=False` (so user can log in).

2. **Login and cookies**
   - `driver.get("https://www.pornhub.com")` → user logs in manually.
   - After Enter, script waits 5 seconds, then:
   - **`save_cookies(driver, cookie_file)`**  
     - Reads `driver.get_cookies()`.
     - Writes a **Netscape HTTP Cookie File** to `cookie_file` (e.g. `logs/pornhub_cookies.txt`):
       - Header: `# Netscape HTTP Cookie File`, `# This file was generated by Selenium`.
       - Each line: `domain\tflag\tpath\tsecure\texpiry\tname\tvalue`  
         (`flag`: TRUE if domain starts with `.`, else FALSE; `secure`: TRUE/FALSE from cookie).

3. **URL extraction (current: Selenium + HTML parse; avoids playlist 404)**
   - **`extract_video_urls_selenium(driver, playlist_url)`**  
     - Navigates to `playlist_url` (e.g. favorites) in the same browser session, then reads `driver.page_source`.
   - **`parse_video_links_pornhub(html, base_url)`**  
     - Parses HTML with BeautifulSoup; selects `a[href*="view_video"], a[href*="viewkey="]`, keeps `href` containing `viewkey=`, builds full URL with `urljoin(base_url, href)`, deduplicates. Returns list of video page URLs.
   - **Previously:** URL extraction used **`extract_video_urls(cookie_file, playlist_url)`** with `yt-dlp --cookies ... --flat-playlist --print url <playlist_url>`. yt-dlp often returns **HTTP 404** on the favorites playlist URL, so that path was replaced by Selenium + HTML parsing.

4. **Download (per video)**
   - **`download_video(video_url, cookie_file)`**
   - **Filename:**  
     `python -m yt_dlp --cookies <cookie_file> --get-filename -o "downloads/%(title)s.%(ext)s" <video_url>`  
     If that fails, no skip-by-filename; else if file exists, skip.
   - **Download:**  
     yt-dlp with cookies and a **format spec that prefers non-HLS** (MP4/get_media) to reduce 404/412 on time-limited HLS segments; fallback to `bestvideo[height<=1080]+bestaudio/best[height<=1080]` if only HLS is available.
   - So: **Netscape cookie file** + **single video page URL** (`view_video.php?viewkey=...`) → yt-dlp does format selection and download.  
   - **404 on “fragment”:** If you see `[download] Got error: HTTP Error 404: Not Found. Retrying fragment 1 (2/10)...`, that is **yt-dlp** failing on a **media fragment** (e.g. HLS/DASH segment) while downloading a **single** video. PornHub's HLS segment URLs are time-limited and often expire (412 Gone). **Mitigation:** update yt-dlp (`pip install -U yt-dlp`) and use ph.py's format preference for non-HLS (MP4/get_media) when available.

5. **Parallel**
   - **`download_videos_parallel(cookie_file, video_urls, max_workers=4)`**  
   - Takes the list of video URLs (from **`extract_video_urls_selenium`** in `main()`), then downloads each with **`download_video(url, cookie_file)`** via `ThreadPoolExecutor(max_workers=4)`.

## 4. How `main.py` Handles PornHub (Precise)

1. **Browser**
   - **Chrome** via Selenium (webdriver-manager), headless optional.

2. **Login and cookies**
   - Same idea as ph.py: open PornHub, user logs in, Enter, wait, then **`save_cookies_netscape(driver, cookie_file)`** producing a Netscape cookie file.

3. **URL extraction**
   - **`extract_video_urls_ytdlp(cookie_file, playlist_url)`**  
   - Same as ph.py: `python -m yt_dlp --cookies <cookie_file> --flat-playlist --print url <playlist_url>`.  
   - Same 404 risk on favorites playlist URL.

4. **Download**
   - **`download_videos_parallel(...)`** (in main.py) uses the same cookie file and per-URL yt-dlp style (format selection and output path are defined in main.py’s download path).

## 5. Handling the 404: Recommended Approach

- **Playlist 404:**  
  Do **not** rely on yt-dlp to extract URLs from the **favorites playlist** URL. Instead:
  1. After login, use the **same browser session** to open the **favorites page** URL (e.g. `users/.../videos/favorites`).
  2. Get **page HTML** (e.g. `driver.page_source` after load, optionally with scrolling for lazy-loaded links).
  3. **Parse HTML** for video links: any `<a href="...">` where `href` contains `view_video` or `viewkey=`. Normalize to full URLs with `https://www.pornhub.com` as base.
  4. Pass the resulting list of **video page URLs** (`view_video.php?viewkey=...`) to the **existing download logic** (yt-dlp with Netscape cookies, same format and output as now).

- **Fragment 404 during download:**  
  Keep using yt-dlp for the actual download; if a fragment 404 persists, it’s likely environment/site/segment expiry; retries are already done by yt-dlp. Optionally surface the error to the user (e.g. “Video X failed after retries”) and continue with other videos.
